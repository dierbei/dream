## 问题
```text
/proc/$$/ns
```

## 回答
```text
/proc/$$/ns 是一个符号链接，指向了当前进程的所有 namespace 目录

。$$ 是 shell 内置的环境变量，表示当前 shell 进程的 PID（进程 ID）。
```

## 问题
```text
为什么这个目录名称 有个 -> 符号，类似于这样cgroup -> cgroup:[4026531835]
```

## 回答
```text
这个符号表明该目录是一个符号链接（Symbolic Link）。

符号链接是一种文件，它包含另一个文件或目录的路径，相当于一个指针。

在Linux系统中，符号链接通常使用ln -s命令创建。
```

## 问题
```text
能不能说说每个文件夹的目录的作用，顺便举个例子？
```

## 回答
```text
在/proc/<pid>/ns目录下，有以下文件夹：
1. cgroup: 该进程的cgroup命名空间，用于控制进程所属的cgroup组。

2. ipc: 进程的IPC命名空间，用于控制进程之间的进程间通信(IPC)。

3. mnt: 进程的挂载命名空间，用于控制进程的文件系统挂载点，包括挂载和卸载文件系统。

4. net: 进程的网络命名空间，用于控制进程的网络接口和路由表，以及网络协议栈的状态。

5. pid: 进程的PID命名空间，用于控制进程的PID，使得该进程只能看到在该命名空间下的进程。

6. user: 进程的用户命名空间，用于控制进程的用户和用户组，以及进程能够访问的文件、目录和设备等资源。

7. uts: 进程的UTS命名空间，用于控制进程的主机名和域名。

例如，/proc/1234/ns/mnt 文件夹中包含了该进程的文件系统挂载命名空间。

如果想要将进程移动到另一个挂载命名空间中，可以使用unshare系统调用，或者使用nsenter命令进入该命名空间。
```
